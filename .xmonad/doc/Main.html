layout' :: ModifiedLayout Rename (ModifiedLayout (ConfigurableBorder Ambiguity) (PerWorkspace (ModifiedLayout (Decoration TabbedDecoration DefaultShrinker) Simplest) (ModifiedLayout SmartSpacing (ModifiedLayout Minimize (ModifiedLayout Maximize (MultiToggle (HCons REFLECTX EOT) (MultiToggle (HCons REFLECTY EOT) (Choose (ModifiedLayout Rename MultiCol) (Choose (ModifiedLayout Rename (Mirror MultiCol)) (Choose Grid Full)))))))))) Window Source #

Custom layoutHook. See XMonad.Config for default layout.

Added:

  • maximize to be able to "temporarily yank the focused window out of the layout to mostly fill the screen" with xK_backslash (see keys').
  • minimize to "make it possible to minimize windows and temporarily removing them from the layout until they are restored" with xK_m and shift + xK_m, respectively (see keys').
  • Ability to "reflect a layout horizontally or vertically" and to toggle those reflections (XMonad.Layout.Reflect and XMonad.Layout.MultiToggle). Key bindings xK_x and xK_y in keys'.
  • Added lessBorders. Since smartBorders (= lessBorders Never) only removes borders if "there is only one screen and only one window" or "a floating window covers the entire screen", we use lessBorders Screen to never draw borders on singleton screens. To differentiate between screen focus, we could use the status bar in the future. noBorders can be used for specific layouts.
  • Added smartSpacing to "add a configurable amount of space around windows", "except when the window is the only visible window on the current workspace".

eventHook' :: Event -> X All Source #

Custom handleEventHook.

Compose event hooks with <+> or a list with composeAll.

Added ewmhDesktopsEventHook to be able to react to

  • _NET_CURRENT_DESKTOP (switching desktops)
  • _NET_WM_DESKTOP (move windows to other desktops)
  • _NET_ACTIVE_WINDOW (activate another window, changing workspace if needed) and fullscreenEventHook to "handle applications that wish to fullscreen using the _NET_WM_STATE protocol".

logHook' :: X () Source #

Custom logHook.

Added ewmhDesktopsLogHook to "notify pagers and window lists, such as those in the gnome-panel of the current state of workspaces and windows".

startupHook' :: X () Source #

Custom startupHook.

Added ewmhDesktopsStartup to "initialize EwmhDesktops and advertise EWMH support to the X server".

main :: IO () Source #

Main entry point.

Possible status bars: xmobar', dzen'.

When using statusBar, the following is already taken care of:

  • Status bar spawned via spawnPipe
  • dynamicLogWithPP added to logHook
  • ppOutput added to PP
  • avoidStruts added to layoutHook
  • manageDocks added to manageHook, docksEventHook to handleEventHook and docksStartupHook to startupHook via docks config modifier.

xmobarIcon :: String -> String Source #

Xmobar icon string with given icon name.

The iconRoot is passed in xmobar'.

Example:

xmobarIcon "myIcon" -> "<icon=myIcon/>"

dzenEvents Source #

Arguments

:: [(String, [(String, [String])])]

dzen events: [ (event1, [ (action1, [option1, option2, ...]), (action2, [...]), ... ]) , (event2, [...]) , ...]

-> String

formatted events

Formatted Dzen events, actions and options for use with command line.

Example:

events = [ ("button1", [("exec", ["xterm", "firefox"])])
         , ("entertitle", [("uncollapse", []), ("unhide", [])])
         , ("button3", [("exit", [])])
         ]

dzenEvents events -> "button1=exec:xterm:firefox;entertitle=uncollapse,unhide;button3=exit"
